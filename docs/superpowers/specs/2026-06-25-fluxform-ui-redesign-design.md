# FluxForm UI 重设计规格

**目标**
将当前 WPF 主界面重构为“批次配置 + 任务队列执行”的双阶段工作台，减少配置与执行职责混杂的问题，同时为视频、音频、图片等类型提供可展开的高级参数配置能力。

**设计原则**
- 配置与执行分离：先配置，再生成任务，再执行。
- 批次优先：用户一次配置一批同类型文件，生成一个批次卡片。
- 高级参数按类型动态显示，默认折叠。
- 执行期间锁定输入与新增任务，避免运行中修改任务集合。
- 第一版任务执行顺序为串行：一次只处理一个任务。

---

## 1. 总体布局

主窗口分为 4 个区域：

1. 顶部全局操作栏
2. 上半区待配置区
3. 下半区任务队列区
4. 底部日志 / 错误详情折叠区

整体采用“两阶段工作流”：

- 阶段 1：添加同类型文件 → 配置输出格式与参数 → 加入任务队列
- 阶段 2：点击开始任务 → 串行执行队列任务 → 查看进度与结果

---

## 2. 顶部全局操作栏

顶部仅放全局行为，不承担当前批次的具体参数设置。

建议按钮：
- 添加文件
- 添加文件夹
- 开始任务
- 停止任务
- 清空队列
- 打开输出目录
- 预设管理

行为规则：
- 当任何任务开始执行后，“添加文件”“添加文件夹”“加入任务队列”相关路径全部禁用。
- 执行完成前，不能新增文件，也不能新增任务。

---

## 3. 待配置区（上半区）

待配置区是新版主流程的核心，用于“先添加文件并配置参数，再生成任务”。

### 3.1 整体结构

待配置区采用左右分栏：
- 左侧 40%：待配置文件列表
- 右侧 60%：当前批次配置面板

### 3.2 左侧：待配置文件列表

职责：让用户确认“当前这批文件是否正确”。

展示信息：
- 当前批次类型（视频 / 音频 / 图片 / 文档）
- 文件数量
- 总大小
- 来源（拖拽 / 手动选择，可选）
- 文件列表

每个文件项展示：
- 文件名
- 原格式
- 文件大小
- 基础属性摘要（按类型显示）

示例：
- 视频：`demo.mov · 1.2 GB · 3840×2160 · 60fps`
- 音频：`song.flac · 38 MB · 48kHz · 2 声道`
- 图片：`cover.png · 4.3 MB · 2400×1600`

支持操作：
- 删除单个文件
- 清空当前批次
- 继续添加同类型文件

### 3.3 同类型限制

同一批次只允许同类型文件。

如果用户拖入或选择了不同类型文件：
- 阻止进入配置态
- 显示明确提示：`当前批次只支持同类型文件，请分别添加视频、音频、图片或文档文件。`

第一版不支持混合类型自动拆分。

### 3.4 右侧：当前批次配置面板

职责：为当前批次设置输出规则与参数。

配置面板分为 4 个区块，自上而下排列：

#### 区块 1：输出目标设置
所有类型通用，默认展开。

字段：
- 输出格式
- 输出目录
- 文件命名规则
- 同名文件处理方式（覆盖 / 自动重命名 / 跳过）

#### 区块 2：快速参数设置
展示该类型最常用的高频参数，默认展开。

视频：
- 编码器
- 分辨率
- 帧率
- 码率

音频：
- 编码器
- 码率
- 采样率
- 声道

图片：
- 输出格式
- 宽高
- 缩放模式
- 质量

文档：
- 输出格式
- 文档导出相关参数（初版保持最小化）

#### 区块 3：高级参数
默认折叠，仅专业用户展开。

视频高级参数：
- 宽高比
- 尺寸模式（拉伸 / 保持比例 / 裁剪）
- 音频处理方式
- 裁剪 / 旋转
- 去元数据
- 自定义附加参数

音频高级参数：
- 音量标准化
- 采样 / 编码附加参数

图片高级参数：
- DPI
- 背景填充
- 透明处理
- 压缩等级

文档高级参数：
- 初版尽量少，避免过重

#### 区块 4：提交与预设
底部动作区：
- 加入任务队列（主按钮）
- 保存为预设
- 恢复默认
- 清空当前配置（可选）

“加入任务队列”是当前批次的正式提交动作，文案保持明确。

### 3.5 待配置区状态

#### 空状态
当用户尚未添加文件时，显示拖拽入口与添加按钮提示：
- 拖入同类型文件开始配置
- 或点击添加文件 / 添加文件夹

#### 已添加但未提交状态
显示文件列表与配置面板。
若必要项未完成（例如未选输出格式、输出目录非法），则禁用“加入任务队列”。

#### 已加入任务队列后
点击“加入任务队列”后：
- 生成一批任务
- 在任务队列中新增一个批次卡片
- 清空待配置区
- 回到空状态，等待下一批输入（前提是任务尚未开始执行）

#### 执行锁定状态
一旦点击“开始任务”：
- 待配置区整体禁用
- 禁止添加文件
- 禁止生成新任务
- 直到全部任务结束后才解锁

---

## 4. 任务队列区（下半区）

任务队列区是执行区与监控区，不承担复杂参数编辑。

### 4.1 队列的结构模型

队列不再以“单任务平铺表格”为主，而是采用：

- 一级：批次卡片（Batch Card）
- 二级：批次内部的小任务卡片（Task Tile）

每次点击“加入任务队列”，即生成一个批次卡片。

### 4.2 批次卡片信息

每个批次卡片显示：
- 批次编号 / 名称
- 类型（视频 / 音频 / 图片 / 文档）
- 任务数量
- 创建时间
- 当前配置摘要
- 批次整体状态
- 批次总进度

示例摘要：
- 视频批次：`MP4 / H.264 / 1080p / 30fps`
- 图片批次：`JPG / 宽 1920 / 质量 85`
- 音频批次：`MP3 / 320kbps / 48kHz`

### 4.3 批次进度

批次卡片内显示：
- 总进度条
- 已完成数
- 失败数
- 未完成数

### 4.4 批次操作

建议支持：
- 展开 / 收起详情
- 重试失败任务
- 删除批次（仅适用于安全场景，如未运行或全部完成/失败后）

如保留全局“开始任务”，则批次卡片无需单独“开始”按钮。

### 4.5 批次内的小任务卡片

展开批次卡片后，显示该批次中的多个小任务卡片。

每个小任务卡片展示：
- 文件名
- 输入格式 → 输出格式
- 当前状态
- 当前进度
- 简短错误信息或成功信息
- 可用操作

成功任务可用操作：
- 打开输出文件
- 打开所在目录

失败任务可用操作：
- 重试
- 查看详情

### 4.6 队列顶部工具条

建议包含：
- 搜索批次 / 搜索文件
- 状态筛选（全部 / 待执行 / 进行中 / 已完成 / 失败）
- 仅看失败项
- 重试全部失败任务
- 清空全部任务

### 4.7 队列底部统计栏

展示：
- 总任务数
- 待执行数
- 进行中数
- 已完成数
- 失败数

示例：
`共 24 项｜待执行 8｜进行中 2｜已完成 11｜失败 3`

---

## 5. 状态与执行规则

### 5.1 任务状态

第一版统一使用 4 种状态：
- 待执行
- 进行中
- 已完成
- 失败

不额外引入“已停止”状态。

### 5.2 执行顺序

第一版采用串行执行：
- 一次只处理一个任务
- 按队列顺序依次执行

后续版本再考虑并行 / 多线程。

### 5.3 点击开始任务后的规则

点击“开始任务”后：
- 队列进入执行态
- 待配置区锁定
- 禁止添加文件
- 禁止新增任务
- 直到当前队列处理完成

### 5.4 点击停止后的规则

点击“停止”后：
- 当前正在执行的任务标记为失败
- 所有尚未执行的排队任务也统一标记为失败
- 已完成任务保持成功

用户视角统一理解为：本次批处理被中断，所有未完成项都算失败。

### 5.5 重试规则

失败任务允许单独重试。

建议支持两种重试入口：
- 单个失败任务重试
- 批次级“重试失败任务”
- 全局“重试全部失败任务”（可选）

重试后进入正常队列顺序继续串行执行。

### 5.6 修改任务规则

第一版不支持在任务队列内直接编辑复杂参数。

如需修改：
- 删除原任务
- 或后续增加“复制配置回待配置区”能力
- 再重新生成任务

当前版本以职责清晰为优先，不在队列区承担复杂配置编辑。

---

## 6. 日志 / 错误详情区

底部区域采用折叠抽屉或标签页形式，默认收起。

建议标签：
- 日志
- 错误详情
- 输出信息

规则：
- 失败任务必须在任务小卡片上先显示简短错误摘要
- 详细错误再在底部详情区中查看
- 日志区不应长期占据主工作区高度

---

## 7. 用户完整交互流程

### 流程 A：创建任务批次
1. 用户拖拽或选择一批同类型文件
2. 文件进入待配置区
3. 系统识别类型并展示可用输出格式与参数
4. 用户设置基础项与高级参数（可选）
5. 点击“加入任务队列”
6. 生成一个批次卡片并清空待配置区

### 流程 B：执行任务
1. 用户点击“开始任务”
2. 系统按队列顺序串行执行任务
3. 执行期间锁定待配置区与新增任务入口
4. 用户在批次卡片与小任务卡片中查看状态与进度

### 流程 C：中断与恢复
1. 用户点击“停止”
2. 当前未完成任务统一标记为失败
3. 用户可对失败任务单独重试，或对批次重试失败项

---

## 8. 第一版范围边界

第一版明确包含：
- 双阶段工作流（待配置区 + 任务队列区）
- 同类型文件批次配置
- 动态媒体参数面板
- 高级参数折叠
- 批次卡片 + 小任务卡片队列
- 串行执行
- 停止即失败策略
- 失败任务重试

第一版明确不包含：
- 多线程 / 并行执行
- 混合类型自动拆批
- 队列内复杂任务参数编辑
- 过重的文档高级参数系统

---

## 9. 验收标准

当以下条件满足时，可认为该 UI 重设计达成目标：

1. 用户能直观理解“先配置，再加入队列，再执行”的主流程。
2. 待配置区与任务队列区职责清晰，没有混淆。
3. 视频 / 音频 / 图片参数按类型动态显示，且高级项默认折叠。
4. 执行期间无法新增文件或任务。
5. 停止后所有未完成项统一进入失败状态。
6. 失败任务可被单独重试。
7. 批次卡片与任务小卡片能清楚反映一批任务及其内部状态。
8. 日志与错误详情可查，但不会长期挤占主工作区。
